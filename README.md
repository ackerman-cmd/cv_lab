# Lab 1 — CV: Car License Plate Detection

Лабораторная работа по обнаружению автомобильных номеров с использованием YOLOv11.

## Задача

Object detection — обнаружение bounding box'ов номерных знаков на фотографиях автомобилей.  
Практическое применение: ANPR-pipeline (Automatic Number Plate Recognition) — первый этап перед OCR.

## Датасет

**[Car License Plate Detection — Kaggle](https://www.kaggle.com/datasets/andrewmvd/car-plate-detection)**

- 433 изображения, 1 класс (`licence`), 471 bbox
- Аннотации в формате Pascal VOC (XML)
- Среднее число объектов на изображение: 1.09
- Разбивка: train 303 / val 64 / test 66

## Метрики

| Метрика | Описание |
|---|---|
| **mAP@0.5:0.95** | Основная. COCO-style, строгая локализация — критична для OCR |
| mAP@0.5 | Стандарт PASCAL VOC |
| Precision | Доля истинных срабатываний |
| Recall | Доля найденных номеров (пропуск критичен для ГИБДД-систем) |
| F1 | Оптимальный порог уверенности |

## Структура экспериментов

### Бейзлайн (раздел 3)

Две модели из семейства YOLOv11 с дефолтными параметрами:
- **YOLOv11n** (~2.6M параметров)
- **YOLOv11s** (~9.4M параметров)

Параметры: `epochs=30, imgsz=640, batch=16`.

### Ablation-исследование (раздел 4b)

Четыре гипотезы, проверялись по отдельности на YOLOv11n за 25 эпох:

| Гипотеза | Конфиг | mAP@0.5:0.95 | Итог |
|---|---|---|---|
| Базовый (контроль) | без изменений | 0.558 | — |
| Увеличение разрешения | `imgsz=800` | 0.553 | Нет эффекта |
| Усиленные аугментации | mosaic + mixup + copy_paste | 0.521 | Хуже |
| AdamW + cosine LR | `optimizer=AdamW, lr0=1e-3, cos_lr=True` | 0.568 | Лучше |

> **Оговорка:** `optimizer=auto` в Ultralytics автоматически переключает SGD→AdamW для малых датасетов, поэтому все ablation-прогоны фактически уже использовали AdamW, что объясняет превосходство контрольного прогона над оригинальным бейзлайном.

### Улучшенный бейзлайн (раздел 4c–f)

| Конфиг | Описание | Результат (yolo11s mAP@0.5:0.95) |
|---|---|---|
| improved_v1 | Все 4 гипотезы сразу: AdamW + imgsz=800 + аугментации + 80 эпох | 0.491 ↓ |
| improved_v2 | Только AdamW + cosine LR, 80 эпох | 0.505 ↓ |

Оба конфига не превзошли бейзлайн. Причина: избыточная регуляризация для 303 train-изображений.

### Fine-tuning с заморозкой backbone'а (раздел 5)

`freeze=10` + improved_v1 конфиг на YOLOv11s: mAP@0.5:0.95 = **0.505** — ниже бейзлайна.  
Вывод: полный fine-tuning предпочтительнее при таком объёме данных.

## Ключевые выводы

1. **Дефолтный конфиг Ultralytics оказался оптимальным** для малого однокласного датасета с крупными объектами.
2. **Агрессивная регуляризация вредит**: mixup + copy_paste + большое разрешение на 303 изображениях размывают представление о точной рамке.
3. **Ablation-прогнозы не всегда переносятся** на длинное обучение: AdamW помогал на 25 эпохах, но не на 80.
4. **Статистическая оговорка**: test-сет 66 изображений, разница ±0.03 mAP находится в пределах шума; направление «улучшения хуже бейзлайна» согласовано по обеим архитектурам.
5. **Надёжный путь к улучшению** — расширение датасета (разные регионы, погода, ракурсы, освещение), а не тюнинг гиперпараметров.
